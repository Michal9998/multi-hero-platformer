ui module

User Interface module for interactive elements.

This module provides reusable UI components such as buttons and sliders that handle their own rendering, state (like hover/locked), and input events.

class ui.Button(x, y, width, height, text, locked=False)

Bases: object

A simple graphical user interface button.

Supports hover visual effects, click detection, and a ‘locked’ state which visually disables the button and ignores clicks (useful for level progression).

draw(surface)

Render the button onto the given surface.

Colors are dynamically evaluated during each frame so they update immediately if the locked state is changed externally.

Parameters:

surface (pygame.Surface) – The target surface to draw the button on.

is_clicked(event)

Check if the button was clicked by the user.

Parameters:

event (pygame.event.Event) – A Pygame event object to evaluate.

Returns:

True if a valid left-click occurred within bounds and the button is unlocked.

Return type:

bool

class ui.Slider(x, y, width, height, min_val, max_val, start_val)

Bases: object

A custom slider UI element to control float values smoothly.

Particularly useful for settings like volume control, allowing the player to click and drag a handle along a track.

draw(surface)

Render the slider track, the filled portion, and the handle.

Parameters:

surface (pygame.Surface) – The target surface to draw the slider on.

handle_event(event)

Process mouse events to handle the dragging mechanics of the slider.

Evaluates mouse clicks, releases, and motion to update the slider’s state.

Parameters:

event (pygame.event.Event) – A Pygame event object to evaluate.

update_handle_pos()

Calculate and update the physical X-coordinate of the handle based on the slider’s current logical value.

update_value_from_pos(mouse_x)

Recalculate the slider’s logical value based on the mouse’s physical X-coordinate.

Parameters:

mouse_x (int) – The current X-coordinate of the mouse cursor.
